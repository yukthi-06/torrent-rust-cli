# User Prompt
> some torrents show 100%, but status is 'Downloading'

# Implementation
- Identified that the issue was purely a CLI presentation bug caused by floating-point rounding. 
- When a torrent reaches `> 99.95%` completion (e.g., missing just a few kilobytes out of gigabytes of data), the `format!("{:.1}%")` macro rounds the value up to exactly `100.0%`.
- Since the math behind the scenes knows the file isn't cryptographically complete, the status correctly remained `"Downloading"`, leading to the confusing `100.0% Downloading` state.
- Fixed this in `crates/cli/src/main.rs` by adding a clamp: if progress is strictly between `99.9` and `100.0`, it is hardcoded to display as `"99.9%"`. The display will now strictly show `100.0%` only when the torrent mathematically hits exactly 100% and transitions to `"Completed"`.
