# BitTorrent Client In Rust - Tracker, Peer Wire Protocol & Download Engine Plan

This plan details the implementation of the core networking and download engine for our BitTorrent client:
1. **Tracker Support (Milestone 9)**: HTTP and UDP tracker announce client to retrieve peer list.
2. **Peer Wire Protocol (Milestone 8)**: Standard TCP handshakes, messages (keep-alive, choke, unchoke, interested, not interested, have, bitfield, request, piece, cancel).
3. **Download Engine (Milestone 10)**: Coordinator managing active peer sessions, downloading piece blocks, verifying piece hashes, and writing blocks to files.

---

## User Review Required

> [!IMPORTANT]
> - **TCP & UDP Networking**: We will implement HTTP and UDP tracker announces using `tokio::net::UdpSocket` and standard TCP sockets for peer communication.
> - **Choking / Interested State**: We will implement simple interest and choking state machine logic on each peer connection.
> - **File I/O**: We will write downloaded blocks directly to the configured target download folder.
