# User Prompt
> Failed: Could not fetch metadata from any of 9 peers

# Analysis
Finding 9 peers is progress (tracker communication works). The issue is that `try_fetch_from_peer` fails for every single peer. The most likely cause is that peers send BitTorrent message IDs we don't handle (e.g., Port/DHT message ID 9), which causes `PeerMessage::read` to return an `Err("Unknown message ID: 9")`, aborting the entire peer attempt.

# Fix
- Update `PeerMessage::read` in `protocol.rs` to gracefully handle unknown message IDs by consuming the bytes and returning a new `PeerMessage::Unknown` variant instead of erroring out.
