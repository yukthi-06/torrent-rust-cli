# User Prompt
> Restore: failed to read magnet:?xt=urn:btih:... : The filename, directory name, or volume label syntax is incorrect. (os error 123)

# Analysis
The daemon persists the magnet URI string in `saved_paths`. On restart, `restore_torrent()` tries to `std::fs::read(path)` on the saved string, but since it's a magnet URI (not a file path), Windows returns OS error 123.

# Fix
Update `restore_torrent()` to detect if the saved path starts with `magnet:?` and route it to `MagnetWorker` instead of trying to read it as a file.
