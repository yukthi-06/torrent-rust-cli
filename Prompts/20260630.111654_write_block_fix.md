# Bug Fix: Piece writing across multiple files

## Issue
When dealing with multi-file torrents, it's common for a downloaded piece (e.g. 16KB blocks) to straddle the boundary between two adjacent files. 
Previously, `write_block` in `crates/daemon/src/engine.rs` checked which file the absolute offset of the *start* of the block belonged to, and then wrote the **entire** block of data into that single file. 

## Fix Implemented
Updated the logic in `write_block` to correctly slice the incoming data and iterate over the files. Now, if a block spans a file boundary, it writes the appropriate bytes up to the end of the first file, then moves to the next file and writes the remaining bytes. This correctly mirrors how `read_piece_from_disk` is correctly implemented for reading pieces across file boundaries.
