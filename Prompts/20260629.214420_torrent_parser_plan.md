# BitTorrent Client In Rust - Torrent Parser & Bencode Library Plan

This plan details the implementation of **Milestone 7: Torrent Parser**. This includes a Bencode parser, SHA1 piece verification model, and support for reading both single-file and multi-file `.torrent` metadata.

---

## User Review Required

> [!IMPORTANT]

> - **Metadata Models**: We will define standard structs for Torrent metainfo: `TorrentMeta`, `InfoDict`, `FileMode` (Single/Multi-file), `InfoHash`, and piece lists.

## Proposed Changes

### Torrent Core / Bencode Parsing

#### [NEW] [crates/core/src/bencode.rs](file:///x:/Projects_X/0_Active/E_Rust/torrent-rust-cli/crates/core/src/bencode.rs)
Implement a Bencode value enum:
```rust
pub enum Bencode {
    Int(i64),
    ByteString(Vec<u8>),
    List(Vec<Bencode>),
    Dict(std::collections::BTreeMap<Vec<u8>, Bencode>),
}
```
Include decoding logic to parse byte streams into `Bencode` values, and encoding logic to serialize them.

#### [NEW] [crates/core/src/meta.rs](file:///x:/Projects_X/0_Active/E_Rust/torrent-rust-cli/crates/core/src/meta.rs)
Define Torrent metadata representation:
- `InfoDict`: Piece length, SHA1 hashes (pieces field), single-file name/length, or multi-files list.
- `TorrentMeta`: Announce URL, Announce-list, Infohash, and info dictionary.

Expose a parser helper `TorrentMeta::from_bytes(bytes: &[u8]) -> Result<Self, anyhow::Error>` which decodes the bencode data, calculates the SHA-1 InfoHash of the info dictionary segment, and populates the models.

#### [MODIFY] [crates/core/src/lib.rs](file:///x:/Projects_X/0_Active/E_Rust/torrent-rust-cli/crates/core/src/lib.rs)
Expose `bencode` and `meta` modules.
