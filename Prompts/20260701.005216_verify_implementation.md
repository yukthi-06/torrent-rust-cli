# Implementation
- Outlined a plan to implement full lifecycle management for the `torrent verify` command.
- Modified `server.rs` to wrap the state in a `TorrentHandle` struct containing the `tokio::task::JoinHandle`.
- Updated `TorrentDownloader::start` in `engine.rs` to spawn peers into a `tokio::task::JoinSet`.
- Updated `Request::Pause` and `Request::Remove` in `server.rs` to call `.abort()` on the `worker_handle`, which drops the `JoinSet` and guarantees all peer connections are cleanly severed and pieces refunded.
- Implemented `Request::Verify` in `server.rs` which aborts the running handle, sets status to `"Checking"`, and spawns a fresh `restore_torrent` process to run the cryptographic disk hash check.
- Stripped the detached `tokio::spawn` from `MagnetWorker::start()` in `magnet_worker.rs` so magnet resolutions can also be aborted cleanly.
