# Goal

Create a modern BitTorrent client consisting of two binaries:

1. torrentd
   - Long-running background daemon
   - Owns all torrent state
   - Handles networking, disk I/O, scheduling, and persistence

2. torrent
   - Foreground CLI client
   - Connects to the daemon over local IPC
   - Never directly manipulates torrent data
   - Simply sends commands and prints results

The project should be structured as a Cargo workspace and written with clean, modular, idiomatic Rust.

# BitTorrent Client In Rust - Workspace Setup & Common Crates Plan

This plan details the first milestone of the BitTorrent client implementation in Rust:
1. **Workspace Setup**: Creating the `Cargo.toml` workspace and basic directory layout.
2. **Common/Core Crates**: Setting up initial definitions in `core`, `config`, `piece`, and `rpc` models.

---

## User Review Required

> [!IMPORTANT]
> - We will structure the workspace according to the specified list under `crates/`.
> - The initial version of the workspace will define the dependencies under a workspace-wide dependency section for consistent versioning.
> - We will use stable Rust with Tokio.
> - The storage will be file-based as requested, but the suggested order says "SQLite storage" under milestone 4 and "file-based persistent storage" under Storage. We will implement file-based persistent storage instead of SQLite, aligning with the "Storage" section of the requirements.
