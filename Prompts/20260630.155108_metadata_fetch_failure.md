# User Prompt
> Failed: Could not fetch metadata from any peer 0

# Analysis
The daemon is successfully finding peers via trackers, but `try_fetch_from_peer` is returning an `Err` for every single peer it connects to. This suggests a protocol error, a timeout, or a peer rejecting the handshake or extended messages. 

We will debug `try_fetch_from_peer` to find where the peers are dropping the connection. Potential causes:
- Peer timeout during `tcp` connection.
- Handshake `info_hash` mismatch.
- Peer doesn't support the extension protocol (bit 43 not set).
- Timeout waiting for `PeerMessage::Extended(0)`.
- Peer drops connection when we send our `PeerMessage::Extended(0)`.
