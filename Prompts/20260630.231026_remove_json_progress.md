# User Prompt
> dont save the percentage completed value in JSON.  On startup, let the app do a hash check and determine the completed percentage

# Implementation
- Refactored `server.rs` to stop reading and writing `downloaded` bytes to `torrents.json`.
- The `load_state` and `save_state` methods now use `Vec<(u32, String)>` instead of `Vec<(u32, String, u64)>`.
- When a torrent is restored on startup via `restore_state`, its initial state `downloaded` value is set to `0` and status is set to `Downloading`. 
- The exact progress and completion status is strictly determined by the cryptographic piece verification in `engine.rs` shortly after.
