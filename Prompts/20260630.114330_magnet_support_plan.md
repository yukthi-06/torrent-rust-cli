# User Prompt
> client program's 'add' function is not accepting magnet hash url . it gives error
> Error from daemon: Failed to read torrent file: The filename, directory name, or volume label syntax is incorrect. (os error 123)

# Analysis
The daemon crashes when given a magnet link because the `add` RPC handler in `server.rs` assumes the input is a local `.torrent` file path and attempts to `std::fs::read` it.

Because magnet links (`magnet:?xt=urn:btih:...`) do not contain the full `.torrent` metadata (e.g. piece sizes, file lists), the client cannot begin downloading the content immediately. It must first connect to peers and download the metadata using the BitTorrent Extension Protocol (BEP 10) and Metadata Extension (BEP 9). 

This is a major feature addition, so an Implementation Plan has been created and presented to the user for approval.
