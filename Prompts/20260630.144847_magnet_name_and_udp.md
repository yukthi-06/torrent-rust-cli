# User Prompt
> magnet link added.
> But torrent name not extracted and also below error
> 
> Magnet Download                     0.0 MB     0.0%       Failed: No peers found from trackers 0

# Analysis
Two issues were identified:
1. Magnet links display as "Magnet Download" because the `dn=` (display name) parameter in the magnet link wasn't parsed or used.
2. The daemon failed to find peers because `TrackerClient::announce_udp` failed to properly strip the `udp://` prefix from the tracker URL, resulting in an invalid socket address `udp:` when splitting by `/`.

# Changes Made
1. **`crates/core/src/magnet.rs`**: Added parsing for the `dn=` parameter, decoding it and storing it in a new `name` field in `MagnetLink`.
2. **`crates/daemon/src/server.rs`**: Used `magnet.name` when setting the display name of the new `TorrentState`. If the `name` is missing, it falls back to the `info_hash_str`.
3. **`crates/tracker/src/lib.rs`**: Updated `announce_udp` to call `tracker_addr_str.trim_start_matches("udp://")` before splitting by `/`, allowing UDP domains to be correctly resolved.
