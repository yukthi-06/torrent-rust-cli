# User Prompt
> Failed: Could not fetch metadata from any of 6 peers. Last errors: 223.178.84.138:6881: deadline has elapsed; 183.82.176.239:6881: deadline has elapsed; 152.57.87.16:41179: deadline has elapsed 0

# Analysis
All peers time out with "deadline has elapsed". This means:
- Tracker discovery works (6 peers found)
- TCP connections are hanging (peers behind NAT, firewalled, or unreachable on advertised port)

The current approach tries peers sequentially with 10s timeouts each, which means 6 peers = 60s wasted before giving up.

# Fix
1. Try peers concurrently (race 5 at a time) using `tokio::select!` or `futures::future::select_ok`
2. Add retry loop that re-announces to trackers after failure
3. Reduce per-peer connection timeout to 5s but increase overall retry window
