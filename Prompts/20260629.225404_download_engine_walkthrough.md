# BitTorrent Client In Rust - Download Engine Walkthrough

I have implemented the core download engine, tracker announce protocol, and peer wire protocol to run the torrent download pipeline.

## Changes Made

### Peer Wire Protocol
- **`crates/peer/src/protocol.rs`**: Implemented TCP-based PWP message serialization and deserialization. Handled handshake exchange, Keep-alive, Choke/Unchoke, Interested/NotInterested, Have, Bitfield, Request, Piece, and Cancel messages.

### UDP Tracker Client
- **`crates/tracker/src/lib.rs`**: Implemented the UDP tracker announce client. It manages the two-phase UDP transaction:
  1. Connection Phase (obtaining standard connection ID).
  2. Announce Phase (sending local port, info_hash, peer_id, download info and receiving IPv4 peer socket lists).

### Download Coordinator & State Wiring
- **`crates/daemon/src/engine.rs`**: Implemented the `TorrentDownloader` coordinator that:
  - Automatically initializes target destination files (single-file size allocation or multi-file layout paths creation).
  - Queries tracker peer lists.
  - Spawns peer stream tasks attempting handshakes, interested state negotiation, block requests (16KB), and disk writes at relative piece offsets.
- **`crates/daemon/src/server.rs`**: Shared state safely using `Arc<Mutex<TorrentState>>` and spawned the download engine worker upon receiving `Request::Add`.
