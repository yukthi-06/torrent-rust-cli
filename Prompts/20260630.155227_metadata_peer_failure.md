# User Prompt
> Failed: Could not fetch metadata from any peer 0

# Analysis
The magnet worker is finding peers from trackers but every `try_fetch_from_peer` call fails. Key issues identified:

1. **Tracker URL handling inconsistency**: `engine.rs` strips `udp://` before calling `announce_udp`, but `magnet_worker.rs` passes the full URL. Meanwhile `announce_udp` was also updated to strip `udp://` — causing potential double-stripping in engine.rs.

2. **Missing tracker error logging**: When a tracker fails, the error is silently swallowed. Need to log these failures.

3. **HTTP fallback missing**: `engine.rs` has logic to try HTTP fallback when UDP fails; `magnet_worker.rs` doesn't.

4. **`https://` trackers not handled**: Many magnet links contain `https://` tracker URLs which are not supported by the raw TCP HTTP client.

# Changes Made
- Improved `fetch_metadata` with better tracker error logging and HTTP fallback logic matching engine.rs
- Increased connection/handshake timeouts
- Added more detailed logging throughout `try_fetch_from_peer` to diagnose failures
