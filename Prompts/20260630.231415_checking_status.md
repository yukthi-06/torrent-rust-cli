# User Prompt
> on startup, while the hash check is going on, if torrent list is run, show the tatus as 'checking'

# Implementation
- Updated the initial `status` assignment in `crates/daemon/src/server.rs` to initialize as `"Checking"` instead of `"Downloading"`.
- This applies to both `restore_torrent` (on daemon startup) and `Request::Add` (when adding a new `.torrent` file).
- The status accurately displays as `"Checking"` during the synchronous disk hashing phase inside `engine.rs` (`verify_pieces`), and is automatically transitioned to `"Downloading"` or `"Completed"` upon completion of the verification.
