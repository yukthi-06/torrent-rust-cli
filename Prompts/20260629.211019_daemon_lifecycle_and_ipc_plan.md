# BitTorrent Client In Rust - Daemon Lifecycle & IPC/RPC Server Plan

This plan describes how we will implement:
1. **IPC Layer**: Platform-native IPC (Unix Domain Sockets on Linux, Named Pipes on Windows) abstracted behind a trait.
2. **RPC Protocol**: Serialized request/response flow using the packet layout: `Version (1 byte) | Length (4 bytes) | Payload (bincode)`.
3. **Daemon Lifecycle**: Long-running async server loop in `torrentd` with graceful shutdown, configuration loading, and signal handling.
4. **CLI Integration**: Connecting the `torrent` CLI to the daemon via IPC to send commands and show responses.

---

## User Review Required

> [!IMPORTANT]
> - **Platform-Native IPC**:
>   - Windows: Uses `tokio::net::windows::named_pipe::ServerNamedPipe` / `ClientNamedPipe`.
>   - Linux/Unix: Uses `tokio::net::UnixListener` / `UnixStream`.
>   - We will define a `Transport` trait to abstract these details.
> - **Graceful Shutdown**: The daemon will listen for termination signals (Ctrl+C, SIGTERM) and shut down the RPC server and state loops cleanly.
