## Changes Made

### RPC Layer & Transport
- **`crates/rpc/src/transport.rs`**: Created cross-platform IPC transport abstraction supporting Named Pipes on Windows (`ServerNamedPipe` / `ClientNamedPipe`) and Unix Domain Sockets on Linux/Unix. Added atomic frame reading/writing helpers.
- **`crates/rpc/src/lib.rs`**: Integrated bincode serialization/deserialization helpers for sending/receiving requests and responses.
- **`crates/rpc/tests/ipc_tests.rs`**: Added a complete integration test asserting cross-platform UDS/Named Pipe request/response exchange.

### Daemon Lifecycle
- **`crates/daemon/src/server.rs`**: Implemented `RpcServer` listener loops for Windows and Unix, supporting connection spawning and request parsing.
- **`crates/daemon/src/main.rs`**: Set up daemon startup logs, graceful shutdown listening via Tokio's `ctrl_c()`, and connection task cleanup.

### CLI Client Integration
- **`crates/cli/src/main.rs`**: Configured CLI commands to initiate IPC connections with `torrentd`, send command requests, and display structured daemon responses (e.g. mock status tables for `list` command, status metrics for `stats`, configuration outputs, version tags).
