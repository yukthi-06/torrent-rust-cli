# BitTorrent Client In Rust - GitHub Actions CI/CD Plan

This plan describes how we will set up the GitHub Actions CI/CD workflow to build, package, and release our Rust binaries for the target platforms.

---

## User Review Required

> [!NOTE]
> - We will support building for the requested targets: Linux x64, Linux ARM64, Windows x64, and Windows ARM64.
> - For cross-compiling to ARM64 targets (especially from Ubuntu runner to Linux ARM64, or Windows runner to Windows ARM64), we will use cargo-cross or specific target options.
> - The workflow will trigger on pushes to `main` and on tag pushes (e.g., `v*`). On tags, it will automatically create a GitHub Release and upload the binaries.

---

## Proposed Changes

### GitHub Actions Configuration

#### [NEW] [.github/workflows/ci.yml](file:///x:/Projects_X/0_Active/E_Rust/torrent-rust-cli/.github/workflows/ci.yml)
Create a workflow containing:
1. **Lint / Check Job**: Runs `cargo check`, `cargo fmt`, and `cargo clippy`.
2. **Build Matrix Job**:
   - OS targets:
     - `x86_64-unknown-linux-gnu` (Linux x64)
     - `aarch64-unknown-linux-gnu` (Linux ARM64)
     - `x86_64-pc-windows-msvc` (Windows x64)
     - `aarch64-pc-windows-msvc` (Windows ARM64)
   - Cross compilation helpers (like `cross-rs/cross` for Linux ARM64 if needed, or Rust target additions).
3. **Upload Artifacts**: Upload built binaries as workflow run artifacts.
4. **Publish Release**: When a version tag like `v1.0.0` is pushed, create a GitHub release and upload the binaries as assets.
