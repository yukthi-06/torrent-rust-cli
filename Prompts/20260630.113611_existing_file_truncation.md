# Bug Fix: Pre-existing Oversized Files

## Issue
Even after fixing the piece download length bounding, files were still appearing larger than expected. 

This happens because the user tested the new code on files that were already partially or fully downloaded by the previous, buggy version of the application. The `initialize_files` routine only set the target length of a file if the file did NOT already exist (`!file_path.exists()`). As a result, files that were already oversized on disk by the previous run were left at their oversized length and were never truncated back down to the correct size.

## Fix Implemented
Updated `initialize_files` in `crates/daemon/src/engine.rs` to enforce `file.set_len(length)` on **all** files at startup, even if they already exist on disk. This will automatically repair and truncate any oversized files leftover from earlier downloads to perfectly match the expected size defined in the torrent metadata.
