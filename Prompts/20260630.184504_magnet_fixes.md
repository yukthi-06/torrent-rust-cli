# Implementation Plan & Actions Taken

### 1. Logging Improvements
- **File:** `crates/daemon/src/main.rs`
- **Action:** Bypassed `tracing_appender::non_blocking` completely and replaced it with a direct, unbuffered `std::fs::File` writer (`torrentd.log`). This guarantees that every log line is flushed to the file immediately, preventing 0-byte log files during sudden crashes or timeouts.
- **Action:** Hardcoded the `tracing_subscriber` filter to `debug` level to avoid silent failures caused by external `RUST_LOG` environment variables.

### 2. Raw Magnet Hash Support
- **File:** `crates/daemon/src/server.rs`
- **Action:** Added logic in `Request::Add` to detect raw 40-character hex strings.
- **Action:** Automatically transformed bare hashes into valid `magnet:?xt=urn:btih:...` URIs and appended reliable public trackers (`tracker.opentrackr.org`, `tracker.openbittorrent.com`) so that the daemon can successfully locate seeders.

### 3. Concurrent Tracker Discovery
- **File:** `crates/daemon/src/magnet_worker.rs`
- **Action:** Refactored the `MagnetWorker::discover_peers` function.
- **Action:** Replaced the highly inefficient sequential UDP/HTTP tracker query loop (which took 15-20 seconds to time out *per broken tracker*, totaling minutes of delay) with a concurrent implementation using `tokio::spawn`.
- **Result:** Tracker discovery now queries all URLs simultaneously and takes a maximum of ~15 seconds total.
