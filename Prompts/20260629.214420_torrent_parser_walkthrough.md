# BitTorrent Client In Rust - Torrent Parser Walkthrough

I have implemented the Torrent Parser and Bencode encoder/decoder.

## Changes Made

### Bencode Library
- **`crates/core/src/bencode.rs`**: Created a recursive-descent Bencode decoder and encoder. Implemented integers, byte strings, lists, and dictionary parsing. Added `find_info_dict_slice` to extract exact bytes of the `info` segment for hashing.
- Added extensive Bencode decoding/encoding tests.

### Torrent Parser & Metadata Models
- **`crates/core/src/meta.rs`**: Created metainfo representations for `.torrent` files, supporting:
  - Multi-file layouts (`TorrentFile` path segment vectors).
  - Single-file layout lengths.
  - Announce URL and Announce-list tier tracker lists.
  - Automatically computing the SHA-1 `InfoHash` of the isolated raw bencoded `info` dictionary.
- Added `TorrentMeta::from_bytes` unit tests with mock bencoded single-file payloads.

### Core Export
- **`crates/core/src/lib.rs`**: Registered and exposed `bencode` and `meta` modules.
