# User Prompt
> application configuration should be loaded from a config file JSON.
> It should contain 
> - download location
> - metadata location
> - data write frequency
> - default list of announce trackers to use

# Implementation
- Rewrote `crates/config/src/lib.rs` and `Cargo.toml` to use `serde_json` and matching `Config` fields.
- Updated `crates/daemon/src/main.rs` to load `config.json` at startup and use `config.data_write_frequency_secs`.
- Updated `crates/daemon/src/server.rs` to pass `config.download_dir`, `config.metadata_dir`, and `config.default_trackers` to the engine and magnet workers.
- Updated `crates/daemon/src/magnet_worker.rs` to use `metadata_dir` instead of the hardcoded `"metadata/"`.
